# FrameWorkers — Short summary

**Note:** The film-style pipeline in the repo is **only an example**; the real target is a **reusable orchestration platform**.

Our **systematic contribution** is a **full-stack, repeatable pattern** for multi-agent work: **planning, execution, extensibility, and human oversight are separated and wired through explicit contracts**, instead of folding everything into ad-hoc scripts or a single monolithic agent. Concretely, a **Director** owns intent and **mutates a layered task stack** (structure, pointer, hooks, batch updates—not informal TODO lists); an **Assistant** provides **one** descriptor-driven path for **all** **sub-agents**, so every capability plugs in the same way; **sub-agents** are **registered and composed dynamically** according to the plan; a **frontend** exposes stack and status so operators can steer and debug; and a **shared workspace** ties memory, files, and logs to **traceable** runs. Together this **systematizes** how tasks are **decomposed, revised, executed, and inspected**—new domains reuse the **same skeleton** and only add or swap **sub-agents**, which is the main reusable artifact beyond any single demo workflow.
